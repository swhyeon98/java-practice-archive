# java-lotto-precourse

로또 미션은 사용자가 입력한 구입 금액에 맞게 자동으로 로또를 발행하고,
당첨 번호 및 보너스 번호와 비교하여 당첨 내역과 수익률을 계산해 주는 콘솔 기반 프로그램이다.

우아한테크코스 프리코스 7기 3주 차 과제로,
**객체를 적절히 분리하고 협력시키는 연습**과 **단위 테스트 기반 구현**을 목표로 한다.

---

## 학습 목표

- 관련된 책임을 묶어 클래스로 분리하고, 객체들이 협력하여 하나의 기능을 완성하도록 설계한다.
- 핵심 도메인(로또 번호, 당첨 결과 등)에 대한 단위 테스트를 작성하여 의도한 대로 동작하는지 확인한다.
- 프리코스 2주 차 공통 피드백을 바탕으로,
    - 메서드 길이(15라인) 및 들여쓰기 depth 제한(2까지 허용)
    - else / switch 미사용
    - 예외 처리 및 에러 메시지 일관성
      등을 최대한 반영한다.

---

## 기능 목록

### 1. 로또 구입 금액 입력

- [x] 구입 금액을 입력 받는다.
    - [x] 구입 금액은 1,000원 단위의 양의 정수여야 한다. 1,000원 미만, 0 또는 음수인 경우 예외를 발생시킨다.
    - [x] 1,000원으로 나누어 떨어지지 않는 경우 예외를 발생시킨다.
    - [x] 숫자가 아닌 값, 공백/빈 문자열 입력 시 예외를 발생시킨다.
- [x] 정상적인 금액 입력 시 구매 가능한 로또 개수를 계산한다.
- [ ] `"{n}개를 구매했습니다."` 문구를 출력한다.

### 2. 로또 자동 발행

- [ ] 구입 금액에 비례해 (금액 / 1,000) 개수만큼 로또를 발행한다.
- [ ] 각 로또는 1~45 범위의 중복되지 않는 6개의 번호로 구성된다.
    - [ ] `Randoms.pickUniqueNumbersInRange(1, 45, 6);`을 사용한다.
- [ ] 각 로또 번호를 오름차순으로 정렬하여 출력한다.

### 3. 당첨 번호 입력

- [ ] 당첨 번호를 쉼표(,)로 구분된 문자열로 입력 받는다.
- [ ] 당첨 번호는 6개여야 한다.
    - [x] 6개가 아닌 경우 예외를 발생시킨다.
- [x] 각 번호는 1~45 범위의 정수여야 한다.
    - [x] 범위를 벗어나는 숫자가 포함된 경우 예외를 발생시킨다.
- [x] 중복된 번호가 존재하는 경우 예외를 발생시킨다.

### 4. 보너스 번호 입력

- [ ] 보너스 번호를 입력 받는다.
- [ ] 보너스 번호는 1~45 범위의 정수여야 한다.
    - [ ] 범위를 벗어나는 경우 예외를 발생시킨다.
- [ ] 기존 당첨 번호와 중복될 수 없다.
    - [ ] 당첨 번호와 중복되는 경우 예외를 발생시킨다.
- [ ] 숫자가 아닌 값, 공백/빈 문자열 입력 시 예외를 발생시킨다.

### 5. 당첨 결과 계산

- [ ] 사용자가 구매한 각 로또 번호와 당첨 번호를 비교한다.
    - [ ] 일치하는 번호 개수를 계산한다.
    - [ ] 보너스 번호 일치 여부를 판단한다.
- [ ] 일치 개수와 보너스 번호 여부를 기준으로 등수를 판별한다.
    - 1등: 6개 번호 일치 / 2,000,000,000원
    - 2등: 5개 번호 + 보너스 번호 일치 / 30,000,000원
    - 3등: 5개 번호 일치 / 1,500,000원
    - 4등: 4개 번호 일치 / 50,000원
    - 5등: 3개 번호 일치 / 5,000원
- [ ] 각 등수별 당첨 개수를 집계한다.
- [ ] 총 당첨 금액을 계산한다.

### 6. 수익률 계산 및 출력

- [ ] 구입 금액과 총 당첨 금액을 이용해 수익률을 계산한다.
    - [ ] 수익률 = (총 당첨 금액 / 구입 금액) * 100
    - [ ] 소수점 둘째 자리에서 반올림한다.
- [ ] 등"수별 당첨 통계를 요구 사항의 출력 형식에 맞게 출력한다.
- [ ] `"총 수익률은 {수익률}%입니다."` 형식으로 수익률을 출력한다.

### 7. 예외 처리 공통 요구 사항

- 사용자가 잘못된 값을 입력할 경우 `IllegalArgumentException`을 발생시킨다.
    - `Exception`이 아닌 `IllegalArgumentException`,`IllegalStateException`등과 같은 명확한 유형을 처리한다.
- 예외 메시지는 항상 `"[ERROR]"`로 시작한다.
    - 예) `[ERROR] 로또 번호는 1부터 45 사이의 숫자여야 합니다.`
- 예외 발생 시 해당 입력 단계부터 다시 입력을 받는다.
- 프로그램 종료 시 `System.exit()`를 호출하지 않고 종료한다.

## 입출력 예시

### 입력 예시

```text
구입금액을 입력해 주세요.
8000

당첨 번호를 입력해 주세요.
1,2,3,4,5,6

보너스 번호를 입력해 주세요.
7
```

### 출력 예시
```text
8개를 구매했습니다.
[8, 21, 23, 41, 42, 43]
[3, 5, 11, 16, 32, 38]
[7, 11, 16, 35, 36, 44]
[1, 8, 11, 31, 41, 42]
[13, 14, 16, 38, 42, 45]
[7, 11, 30, 40, 42, 43]
[2, 13, 22, 32, 38, 45]
[1, 3, 5, 14, 22, 45]

당첨 통계
---
3개 일치 (5,000원) - 1개
4개 일치 (50,000원) - 0개
5개 일치 (1,500,000원) - 0개
5개 일치, 보너스 볼 일치 (30,000,000원) - 0개
6개 일치 (2,000,000,000원) - 0개
총 수익률은 62.5%입니다.
```
